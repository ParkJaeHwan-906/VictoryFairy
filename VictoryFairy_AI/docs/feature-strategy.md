# 기능 전략

각 기능의 방향과 근거. 상세 설계는 `/Users/hwannee/Downloads/TalkFile_ai-filter-pipeline-design.md.md` 참조.

## 1. 검열 (validation)

- **정규화 우선** — `병신 → ㅂㅕㅇㅅㅣㄴ / ㅄ` 같은 회피 표기는 학습만으로 못 잡으므로, 룰 기반 전처리로 표면형을 복원한 뒤 매칭한다.
- **다중 뷰 매칭** — 단일 정규화는 한글 사이에 낀 기호·숫자(`씨@발 → 씨a발`)를 놓친다. 그래서 한 문장을 **6개 뷰(원문·정규화·압축·한글·영어·키보드)로 변환해 각각 매칭**한다. `한글` 뷰가 한글 사이 라틴/숫자 노이즈를 제거해 `씨발`로 복원하고, `영어` 뷰가 로마자(`sibal`)를 격리하며, `키보드` 뷰가 한/영 키 미전환 표기(`tlqkf → 시발`)를 두벌식 자판으로 되돌린다. 트레이드오프: 뷰가 단어 경계를 없애 비정상 혼용 표기에서 드물게 오탐(정상 문장에선 미발생) → `exceptions.json`으로 보정. 키보드 뷰는 초성 오탐이 커서 **완성 음절 욕설만** 매칭.
- **룰/정규식 + 사전** — `banned_words.json`(카테고리별 비속어, 초성 욕설 포함) + `exceptions.json`(오탐 방지 whitelist).
- **향후 캐스케이드**(설계 방향) — 룰/인코더 분류기(1차) → 회색지대만 sLLM(2차). 현재는 룰/정규식 단계.

## 2. 형태소·개체명 추출 (analysis)

- **Kiwi + NER 하이브리드** — 명사·동사는 Kiwi(형태소, 결정론적·정확), 이름·지명·기관·날짜는 NER(개체명).
  - Kiwi 단독의 한계: 고유명사(NNP)로 사람/지명/기관을 구분 못함 → NER의 PS/LC/OG/DT로 해결.
- **원문 기준** — 추출은 검열 정규화본이 아니라 원문으로 수행(표면형 보존).
- **문장 분리** — 한 줄에 여러 문장이 있을 수 있어 Kiwi `split_into_sents`로 분리 후 처리.

## 3. 사전 후처리 (gazetteer)

기성 NER 모델의 지식은 학습 시점에 고정되므로, **사전으로 모델 밖에서 보정**한다(학습 불필요).

- **등록 인명/기관 whitelist** — 모델이 놓친 신규·도메인 개체 강제 추가(예: `SSG 랜더스`).
- **성씨 교차검증 + 오탐 blocklist** — 인명형 일반명사(`김밥`) 억제, 한 글자 인명 노이즈 제거.
- **겹침 정리** — 등록 기관이 커버하는 span 안의 조각(`SSG 랜더스` 속 `랜더스`) 제거.
- 파일만 고치면 반영: `analysis/core/data/*.json` (사용법은 그 디렉토리 README.md).

## 4. 인명 정규화·집계

수집·분석 목적이라 재현율만큼 **정밀도·정규화**가 중요(잘못 뽑힌 인명이 DB 오염).

- **별칭 사전 + 부분문자열 클러스터링** — `길동 → 홍길동` 통합. 보수적으로(한 글자 제외) 오병합 방지.
- **집계 스키마** — `{정규화_인명, 표면형_리스트, 출현_문장_id, 빈도}`.
- **한계** — 동명이인은 문자열 집계로 구분 불가(향후 엔티티 링킹 필요).

## 5. 미해결·향후 과제

- `대박 → 박` Kiwi 과분할 → Kiwi 사용자 사전 미도입.
- 검열 인코더/sLLM 캐스케이드(설계 M2~M3) 미착수.
- NER 파인튜닝(도메인 인명 정밀도) — 사전으로 부족할 때 착수.
